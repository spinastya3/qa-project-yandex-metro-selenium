# Автотесты для сервиса Яндекс.Метро (В рамках обучения Яндекс.Практикум)

Проект содержит автоматизированные UI-тесты для проверки ключевой функциональности сервиса Яндекс.Метро: выбор города, поиск станций и расчет времени маршрута.

## 🛠 Технологический стек
* **Java 11**, **JUnit 4**
* **Selenium WebDriver** (UI-автоматизация)
* **Maven** (сборка и управление зависимостями)
* **Allure Report** (профессиональная отчетность)
* **WebDriverManager** (автоматическая конфигурация драйверов)

## 🏗 Архитектура проекта (POM)
Проект реализован с использованием паттерна **Page Object Model**, что обеспечивает чистоту тестов и удобство поддержки кода:
* `BasePage` — содержит общие методы поиска и универсальные ожидания.
* `MetroHomePage` — описывает локаторы и методы взаимодействия с элементами страницы.
* `BaseTest` — базовая настройка жизненного цикла тестов и драйвера.
* `FactoryDriver` — кастомное правило JUnit для гибкого управления браузерами.
* `MetroTestData` — слой тестовых данных (Data Driven подход).

## ⚡ Особенности реализации (Handling Flaky Elements)
В процессе работы с динамическим фронтендом сервиса была решена проблема «мигающих» (flaky) тестов:
* **Стабилизация саджестов**: Реализовано ожидание видимости блока подсказок (`data-block='station-suggest'`) перед взаимодействием.
* **Обход некликбельных заголовков**: Использована логика двойного нажатия `Keys.DOWN` для точного выбора станции в списке подсказок.
* **Фиксация фокуса**: Применен метод двойного клика для активации капризных полей ввода, что обеспечило стабильность прохождения тестов.
